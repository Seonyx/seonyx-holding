## Local Development Setup

### Prerequisites
- .NET Framework 4.5 SDK
- SQL Server or SQL Express
- Visual Studio Code
- Git

### Setup Steps

1. **Clone the repository**
   ```bash
   git clone https://github.com/yourusername/seonyx-holding.git
   cd seonyx-holding
   ```

2. **Create database**
   ```bash
   # Create a new database called 'Seonyx' in SQL Server
   # Run the schema creation script
   sqlcmd -S localhost -d Seonyx -i Database/schema.sql
   
   # Run the seed data script
   sqlcmd -S localhost -d Seonyx -i Database/seed-data.sql
   ```

3. **Configure connection string**
   ```bash
   cd Seonyx.Web
   cp Web.config.template Web.config
   # Edit Web.config and update connection string with your SQL Server details
   ```

4. **Restore NuGet packages**
   ```bash
   nuget restore
   ```

5. **Build and run**
   ```bash
   # Using Visual Studio Code or Visual Studio
   # Build the solution
   # Run with IIS Express
   ```

6. **Access the site**
   - Public site: `http://localhost:port/`
   - Admin area: `http://localhost:port/admin`
   - Default admin credentials in Web.config

## Initial Commit Workflow

```bash
# Initialize git repository
git init

# Add all files
git add .

# Create initial commit
git commit -m "Initial commit: Seonyx Holdings website structure

- ASP.NET MVC 5 project setup
- Entity Framework models for CMS
- Database schema and seed data
- Complete documentation
- Controllers and views structure
- Admin area foundation"

# Add remote repository
git remote add origin https://github.com/yourusername/seonyx-holding.git

# Push to GitHub
git branch -M main
git push -u origin main
```

## Branch Strategy

### Main Branches
- `main` - Production-ready code
- `develop` - Development branch for ongoing work

### Feature Branches
Create feature branches for new features:
```bash
git checkout -b feature/author-profiles
# Work on feature
git commit -m "Add author profile pages"
git push origin feature/author-profiles
# Create pull request to merge into develop
```

### Hotfix Branches
For urgent production fixes:
```bash
git checkout -b hotfix/contact-form-spam
# Fix issue
git commit -m "Fix contact form spam protection"
git push origin hotfix/contact-form-spam
# Merge directly to main and develop
```

## Recommended Commit Message Format

```
<type>: <subject>

<body>

<footer>
```

**Types:**
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation changes
- `style`: Formatting, missing semicolons, etc
- `refactor`: Code restructuring
- `test`: Adding tests
- `chore`: Maintenance tasks

**Examples:**
```
feat: Add author biography pages

- Created Author model and controller
- Added biography display view
- Integrated with literary agency section

Closes #12
```

```
fix: Resolve contact form validation

- Added server-side email validation
- Fixed honeypot anti-spam field
- Improved error messages

Fixes #45
```

## GitHub Actions (Future Enhancement)

Create `.github/workflows/deploy.yml` for automated deployment:

```yaml
name: Deploy to Production

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v2
    
    - name: Setup .NET
      uses: actions/setup-dotnet@v1
      with:
        dotnet-version: 4.5.x
    
    - name: Restore dependencies
      run: nuget restore
    
    - name: Build
      run: msbuild /p:Configuration=Release
    
    - name: Deploy via FTP
      uses: SamKirkland/FTP-Deploy-Action@4.0.0
      with:
        server: ${{ secrets.FTP_SERVER }}
        username: ${{ secrets.FTP_USERNAME }}
        password: ${{ secrets.FTP_PASSWORD }}
        local-dir: ./Seonyx.Web/bin/Release/
```
